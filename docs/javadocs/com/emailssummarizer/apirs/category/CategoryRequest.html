CategoryRequest record

public record CategoryRequest( String name, String code, String description )

Immutable DTO carrying the data required to create or update a Category.

Used as the request body for POST /categories and PUT /categories/{code}. On update, the code component is ignored by the service layer — the path variable is used as the lookup key and the code itself remains immutable.

Validation of individual fields (non-blank name, unique code) is performed by CategoryService, not at the DTO level.

See also: CategoryService

Record Components

name String
The human-readable display name for the category; must not be null.
code String
The unique business key for the category.
Must not be null or blank on creation; ignored on update.
description String
An optional human-readable description; may be null.
