GitHubOpaqueTokenIntrospector class

@Component
public class GitHubOpaqueTokenIntrospector implements OpaqueTokenIntrospector

Validates GitHub opaque access tokens and assigns Spring Security granted authorities.

Calls GET https://api.github.com/user with the Bearer token to confirm validity and retrieve the authenticated user's GitHub login. The login is matched (case-insensitively) against three independent allow-lists to assign ROLE_READ, ROLE_EDIT, and/or ROLE_DEL.

Allow-lists are injected from app.readers, app.editors, and app.deleters application properties, which are bound from environment variables READERS_GITHUB_LOGINS, EDITORS_GITHUB_LOGINS, and DELETERS_GITHUB_LOGINS respectively. Each variable is a comma-separated list of GitHub login names. Roles are independent — a user must appear in each list separately to hold multiple roles.

Warnings are logged at startup if any allow-list is empty, since this would deny all requests for the corresponding HTTP methods.

See also: ResourceServerConfig

Method Summary

@Override public introspect OAuth2AuthenticatedPrincipal
@Override
public OAuth2AuthenticatedPrincipal introspect(String token)
Introspects the given opaque GitHub token by calling the GitHub user-info endpoint.
Sends a GET https://api.github.com/user request with the token as a Bearer credential. On success, extracts the GitHub login and builds a GitHubPrincipal with the applicable granted authorities based on the configured allow-lists.
@param token — the raw GitHub access token to validate; must not be null
@return an OAuth2AuthenticatedPrincipal representing the authenticated GitHub user with zero or more of ROLE_READ, ROLE_EDIT, ROLE_DEL granted authorities
@throws BadOpaqueTokenException if the token is invalid, expired, or the GitHub user-info endpoint returns a non-200 response, or if the response does not contain a login field
