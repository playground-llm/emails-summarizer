GitHubOpaqueTokenIntrospector class

@Component
public class GitHubOpaqueTokenIntrospector implements OpaqueTokenIntrospector

Validates GitHub opaque access tokens and assigns Spring Security granted authorities based on roles stored in the database.

Calls GET https://api.github.com/user with the Bearer token to confirm validity and retrieve the authenticated user's GitHub profile (login, id, name, avatar_url).

The GitHub login is then passed to UserService#findOrRegister, which either returns the user's existing roles from the ROLES table or — if this is the user's first login — creates a new entry in USERS and assigns ROLE_READ automatically.

See also: ResourceServerConfig, UserService

@Override
public OAuth2AuthenticatedPrincipal introspect(String token)
Introspects the given opaque GitHub token by calling the GitHub user-info endpoint.
Sends a GET https://api.github.com/user request with the token as a Bearer credential. On success, extracts the GitHub login and delegates to UserService#findOrRegister to obtain the applicable Spring Security roles. New users receive ROLE_READ automatically on their first authentication.
@param token — the raw GitHub access token to validate; must not be null
@return an OAuth2AuthenticatedPrincipal representing the authenticated GitHub user with the roles stored in the database
@throws BadOpaqueTokenException if the token is invalid, expired, or the GitHub user-info endpoint returns a non-200 response, or if the response does not contain a login field
