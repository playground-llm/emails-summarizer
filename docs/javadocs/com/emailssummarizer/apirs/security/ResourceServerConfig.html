public class ResourceServerConfig

Spring Security configuration for the OAuth2 Resource Server.

Configures a stateless security filter chain that:

  • Validates incoming Bearer tokens via GitHubOpaqueTokenIntrospector.
  • Enforces role-based access control on all category and message endpoints.
  • Permits POST /oauth2/token and the H2 console without authentication.
  • Applies CORS rules that allow only the known Vue UI origins.

Session management is set to STATELESS — no HTTP session is created or used; every request must carry a valid Bearer token (except permit-all paths).

See also: GitHubOpaqueTokenIntrospector

public @Bean securityFilterChain SecurityFilterChain

public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception
Builds and returns the primary SecurityFilterChain for the application.
Authorization rules applied in order:
  1. /h2-console/** and POST /oauth2/token — permit all (no token).
  2. DELETE /categories/**, DELETE /messages/** — requires ROLE_DEL.
  3. POST /categories, POST /messages, PUT /categories/**, PUT /messages/** — requires ROLE_EDIT.
  4. GET /categories, GET /messages — requires ROLE_READ.
  5. All other requests — requires authentication (any valid token).
@param http — the HttpSecurity builder provided by Spring Security
@return the configured SecurityFilterChain
@throws Exception if the security configuration cannot be built
