OAuthController class

@RestController @RequestMapping("/oauth2")
public class OAuthController

Proxies the GitHub OAuth2 Authorization Code → access_token exchange.

GitHub's token endpoint does not emit CORS headers, so a browser cannot call it directly. The UI sends the authorization code to POST /oauth2/token on this controller, which performs the server-to-server exchange and returns only the access_token to the browser — keeping the client_secret on the server at all times.

This endpoint is permitted without authentication (no Bearer token required) so that the UI can obtain a token before it has one.

Request / response contract:

POST /oauth2/token  Content-Type: application/json
{ "code": "...", "redirectUri": "..." }

200 OK  Content-Type: application/json
{ "accessToken": "gho_..." }

public @PostMapping("/token") exchangeToken TokenResponse
@PostMapping(value = "/token", consumes = APPLICATION_JSON, produces = APPLICATION_JSON)
public TokenResponse exchangeToken(@RequestBody TokenRequest request)
Handles POST /oauth2/token requests to exchange a GitHub authorization code for an access token.
Calls https://github.com/login/oauth/access_token server-side with the configured client_id and client_secret, then extracts and returns the access_token from GitHub's response. The client_secret is never exposed to the browser.
@param request — the request body containing the authorization code and the redirectUri; must not be null
@return a TokenResponse containing the GitHub access token
@throws RuntimeException if GitHub does not return an access_token (e.g. the code is invalid, expired, or already used)
