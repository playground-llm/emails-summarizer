UserService class

@Service
public class UserService

Service that manages application user registration and role retrieval.

On every successful GitHub token introspection, the security layer calls findOrRegister with the authenticated user's GitHub profile data. The method implements a "find-or-create" pattern:

  • If the user's login is already present in the USERS table, the user's current roles are loaded from the ROLES table and returned.
  • If the login is not found, a new AppUser row is inserted and a single UserRole row granting ROLE_READ is created automatically.

All database operations run within a single transaction to prevent partial writes.

See also: AppUserRepository, UserRoleRepository

Method Summary

public findOrRegister List<String>
@Transactional
public List<String> findOrRegister(String login, Long githubId, String name, String avatarUrl)
Returns the roles for the given GitHub user, registering them with ROLE_READ if this is their first login.
The entire operation is transactional. If the user already exists, only a SELECT on the ROLES table is issued. If the user is new, an INSERT into USERS and an INSERT into ROLES are issued atomically.
@param login — the lower-cased GitHub login; must not be null
@param githubId — numeric GitHub user ID extracted from the introspection response; may be null
@param name — display name from GitHub; may be null
@param avatarUrl — GitHub avatar URL; may be null
@return a non-null, non-empty list of Spring Security role strings (e.g. ["ROLE_READ"]); never empty because every registered user holds at least ROLE_READ
